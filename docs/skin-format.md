# 皮套格式草案（v0）

> 格式尚未定稿，调整时同步修改 `app/games/avatar_lobby/skins.py` 里的校验和 `FORMAT_VERSION`。

## 当前接受的文件

| 项 | 约定 |
|------|------|
| 文件类型 | 单张 PNG 或 WebP（按魔数校验，不看扩展名） |
| 大小上限 | 2 MB |
| 朝向 | 贴图默认朝右；向左移动时由前端水平翻转 |
| 建议画布 | 4:3 横画幅（683×512），透明背景；旧版 512×512 仍可导入 |

## 皮套类型（manifest.kind）

| kind | 含义 |
|------|------|
| `plain` | 整张贴图当作精灵直接绘制（默认，兼容旧皮套） |
| `uv` | 按下方 UV 版式分部位绘制，四肢跟随关节动画 |

## UV 版式（kind = "uv"，LAYOUT_VERSION = 6）

新版贴图固定 **683×512**（**4:3** 横画幅）：左侧 512px 为部位槽，**右侧留白**；角色骨架约 **8 头身**。每个部位有：

| 字段 | 含义 |
|------|------|
| `rect` | **读取范围**（含留白；四肢留白加大，可画臂甲等） |
| `coreRect` | **推荐绘制区**（模板上的色块与骨骼落在此处） |
| `code` | 模板短码（如 `RA1`） |

槽与槽之间留约 4px 空隙作线分割。留白范围与 `rect` 一致（透明底）。

版式的唯一权威定义在 `static/js/uv-layout.js`。页内「下载模板」由 `uv-template.js` 生成绘画参考图（关节节点、矩形骨骼连线、分色标注）；「导入完整 UV」按各部位 `rect` 拆槽解析。

| 部位 | 短码 | 读取范围 `rect` | 推荐世界尺寸 |
|------|------|------|------|
| 头部/头发（整身画布） | HD | [4, 4, 250, 250] | safe ≈ 18×15 |
| 身体 | BD | [262, 4, 246, 250] | 22×26（推荐区靠上，下留白可画斗篷） |
| 右手臂·上/下 | RA1/RA2 | [4/130, 258, 122, 118] | 7×15 / 7×16 |
| 左手臂·上/下 | LA1/LA2 | [256/382, 258, 122, 118] | 7×15 / 7×16 |
| 右腿·上/下 | RL1/RL2 | [4/130, 380, 122, 118] | 8×16 / 8×17 |
| 左腿·上/下 | LL1/LL2 | [256/382, 380, 122, 118] | 8×16 / 8×17 |

朝右时：`RA`/`RL` = 角色右侧，`LA`/`LL` = 角色左侧。髋约 y=3，站立脚底约 y=36。

读取区大于推荐区时，舞台把 `coreRect` 对齐到该比例，整份 `rect` 同比例外扩，
因此留白里的臂甲/长发会露在轮廓外，基础体型不变。

头部/头发槽绘制为覆盖角色的 72×72 图层（`drawRect`），编辑器黄框为原本头部轮廓（`safeRect`）；透明内容可越过该框延伸，适合长发、帽子、披风。肢体「上边」朝向躯干一侧，绘制时随关节旋转。

旧版 256×256 UV 皮套仍按 `LEGACY_PARTS` 读取。
旧版 **512×512 右栏模板**（左侧大头槽 + 右侧 BODY/FAU…）由运行时 `UVLayout.resolveParts` 自动识别为 `SQUARE_V1_PARTS`，无需重新导出。
新皮套请使用 **683×512** 模板（页内「下载模板」）。

## 存储布局

每个皮套一个目录，方便以后扩展成多贴图 / 动画帧：

```
var/uploads/skins/
└── <skin_id>/              # 12 位十六进制随机 id
    ├── texture.png         # 或 texture.webp
    └── manifest.json
```

`manifest.json` 字段：

```json
{
  "id": "a1b2c3d4e5f6",
  "name": "皮套名称",
  "uploader_id": "上传者 uid",
  "texture": "texture.png",
  "kind": "plain",
  "height_scale": 1.0,
  "format_version": 0,
  "created_at": 1752723600
}
```

`height_scale` 是以脚底为固定点的纵向身高比例，允许范围为 `0.8`–`1.5`；
旧皮套缺少该字段时按默认值 `1.0` 渲染。

## 预留的扩展方向

- 多部件 / 动画帧：目录里加 `walk_01.png` 等文件，manifest 增加帧表
- 行走 / 跳跃动作定义：manifest 增加 `animations` 字段
- 格式升级时 `format_version` 递增，旧皮套按版本号做迁移或兼容渲染
